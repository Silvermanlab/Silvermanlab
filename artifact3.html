Databases Artifact

Artifact Description

This artifact is an Android-based Event Management Application originally developed in CS-360: Mobile Architecture & Programming. The application allows users to create accounts, log in, and manage events through a local SQLite database, supporting full CRUD (Create, Read, Update, Delete) functionality. The application also includes optional SMS notifications for event reminders. This system supports user-centered decision-making by enabling individuals to organize, track, and manage important events efficiently through a structured and accessible interface.

Enhancements Implemented

To improve this artifact for my ePortfolio, I focused on optimizing database design, improving data integrity, enhancing performance, and strengthening overall system reliability. I refactored the SQLite database schema to improve structure and organization and strengthened input validation to prevent invalid or incomplete data from being stored. Data retrieval queries were optimized to enhance application performance, and data handling between the user interface and database layers was improved. In addition, I refactored database-related code for better readability and maintainability and added logging to support debugging and system monitoring. Enhancements were also made to the event management interface to improve validation, error handling, and overall user experience. For example, input validation and error handling were implemented to ensure that required fields are completed and that invalid inputs do not cause application failures.

Enhanced Code

These enhancements resulted in a more efficient, reliable, and scalable application. They also improved the user experience by ensuring accurate data entry and faster retrieval of event information. A key design trade-off in this enhancement process was balancing query performance with code readability and maintainability. While more complex queries could improve performance further, I prioritized structured and readable code to ensure long-term scalability and ease of maintenance. Additionally, input validation improvements reflect a security-focused approach by reducing the risk of invalid or potentially harmful data being processed by the system.
The original database implementation provided basic CRUD functionality but lacked input validation, structured schema constraints, and performance optimization. The enhanced version improves reliability and maintainability by adding input validation to ensure data integrity, structuring the database schema with appropriate constraints, incorporating logging for debugging and monitoring, and optimizing queries for faster and more efficient data retrieval
